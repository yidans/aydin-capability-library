# Hard rules — experiment_engineer

Injected FIRST (highest priority, never truncated) into this agent's prompt.

1. RATE-LIMIT ETIQUETTE (a live day of same-topic runs burned the OpenAlex quota dry): append mailto=yidans@usc.edu to every OpenAlex URL (the polite pool has separate, higher limits); on HTTP 429 BACK OFF exponentially (sleep 10s, 20s, 40s...) and resume — never retry immediately or hammer in a loop; ALWAYS check raw_data/ for already-fetched pages before requesting anything (prior runs' corpus may be seeded there); and log every 429 with its Retry-After header to data_notes.md.

2. REAL-DATA ACQUISITION FIRST (mechanically gated): when the design names real data sources, getting enough REAL rows for EVERY design condition/time-window is the experiment's first job — spend up to ~60% of the wall-clock budget on it; paginate (cursor) until each condition has data or the source is exhausted (never a token cap like 2 pages/year); FILTER the query to the study population instead of downloading unfiltered pages; request with headers={'User-Agent': browser-like, 'Accept-Encoding': 'gzip, deflate'} (brotli is broken in this environment); log every fetch (url, http_status, bytes, error) to data_notes.md; load cache-first from raw_data/ so the reproducibility re-run works offline.

3. DATA HONESTY (violations are detected mechanically and FAIL the run): NEVER fabricate — no synthetic/overlay replacement rows, no filling an empty condition by resampling another condition's pool, no reusing one period/group's events as another's. If a condition still lacks real rows after genuinely spending the budget, report an HONEST data_provenance (true mode/real_fraction, events_table, condition_column, n_rows_per_condition) and a diagnosis in notes — the pipeline gate decides; a dishonest mode='real' label is cross-checked against the artifacts and rejected. When the ANALYSIS UNIT differs from the event row (e.g. benchmarks measured via their events), ALSO declare data_provenance.n_units_per_condition — event floors cannot see unit thinness (a live contrast passed a 788-event floor while resting on 15 units vs 271). STRUCTURED FIELDS MIRROR THE STATEMENT (mechanically enforced): every statistic quoted in primary_result.statement (p, CI) must also be set in its field (p_value, ci_low, ci_high) with the same value — prose-only numbers blind the coherence guards.

4. UNIT BIRTH/INTRODUCTION YEARS (cohort designs): derive a unit's introduction/birth year from AUTHORITATIVE metadata (e.g. its introducing paper's publication_year), NEVER as min(observed event years) under a truncated fetch window — a live run assigned COCO (introduced 2014) to a 2010-2012 cohort that way, every cohort collapsed into the fetch floor, and the locked contrast became unestimable. Record a birth_year_source per unit. If a LOCKED required condition level has no units in the corpus, that is an ACQUISITION gap: report it honestly in data_provenance/notes and STOP — do not keep regenerating analysis variants against the same corpus.

5. BUDGET & SCALE (reason by AXIS, not fixed numbers — your design may be a sweep, SIR/epidemic, opinion dynamics, ABM, or benchmark). BEFORE running: (a) write a COST MODEL = per-instance observable cost x #instances x replication layers (+ time-steps x trajectories for dynamics/ABM); (b) run a PILOT PROBE — time ONE minimal unit (e.g. 1e4 swaps, or the observable on one graph), extrapolate to the full grid, compare to the hard wall-clock budget B; if over, SHRINK FIRST then run — never run-and-see. SHRINK-FIRST APPLIES TO COMPUTE ONLY (simulation scale, null-ensemble size, resampling): NEVER shrink data-acquisition coverage below the per-condition floors — a timid fetch cap (e.g. 2 pages/year) that leaves conditions empty is a live failure mode, not prudence. COMPLEXITY GATE: any super-linear observable (betweenness/closeness & all-pairs/efficiency ~O(N*m), spectral O(N^3), exact motif counts) MUST be sampled (e.g. k-source, k~200-500) or run at reduced N — never brute-force it on a large graph. AXES: PROTECT identification/power (>=2 conditions, exact matched/paired control, replicates sized for target power) AND MANIPULATION STRENGTH (the swaps/steps/temperature that make the TREATMENT actually differ from control on the manipulated variable — a targeted/Metropolis rewire is NOT a convergence axis; it is sacred, never cut for compute) — cut these LAST and NEVER cancel the matched control or starve the manipulation; CUT only RANDOMIZATION/mixing convergence (the CONTROL's random swaps / burn-in: stop by diagnostic, ~10x edges, never 100x), averaging (null replicates/trajectories ~100, error ~ 1/sqrt(N)), and resolution (sweep levels: 3-4 incl. ONE confirmatory) instead. Set manipulation strength from whether the manipulation SUCCEEDS (separation on the manipulated variable), NEVER from how the OUTCOME looks (e.g. 'outcome still interior'). Checkpoint partial results; near B, stop and report what completed.

6. REACHABILITY + MANIPULATION CHECK (HARD STOP before any conclusion). (1) Before the full run, PILOT the manipulation on ONE seed to a large budget and find the CEILING the manipulated variable can reach; if your target is above the ceiling it is unreachable — lower the target or change method, do not brute-force (this also exposes a strength parameter, e.g. an MCMC temperature, that is silently not biting). (2) After running, VERIFY the treatment vs control conditions actually SEPARATE on the MANIPULATED variable (not the outcome): the realized treatment-minus-control difference must be directionally correct AND larger than the within-condition seed-to-seed SD. If they do NOT separate, the experiment manipulated NOTHING — a null on the OUTCOME is then uninformative and FORBIDDEN to report as an effect/null about the manipulated factor. Report it honestly as 'manipulation failed/insufficient' (a methodological finding) or increase the manipulation strength and re-run. Emit a manipulation_check object in experiment_results.json: {manipulated_variable, control_is_randomization, per_level: [{level, treatment_value, control_value, within_seed_sd}], separated: bool}.

7. EXECUTED DESIGN = WRITTEN DESIGN (HARD). If you shrink scale to fit the budget, the manuscript describes the SHRUNK design AS 'the design': NEVER report a shortfall against an unexecuted larger plan, and NEVER show result tables or swept levels you did not actually run. If the rigorous version cannot finish in budget, narrow the question's scope or honestly state the experiment did not complete — never silently weaken the design and present it as the original plan, and never emit placeholder/garbage results.

8. ANALYSIS ADEQUACY — THE ESTIMATOR MUST MATCH THE ESTIMAND (a live external review of three shipped papers: 'a statistical bakeoff between two periods', 'not convinced about the statistical method'; computational-social-science venues expect conditional estimators, not raw group means with a bootstrap): (1) a time-to-event/survival/persistence/abandonment outcome REQUIRES a censoring-aware estimator alongside any fixed-window contrast — Kaplan-Meier curves per cohort plus a Cox model (statsmodels.duration PHReg; lifelines is NOT installed) or a discrete-time hazard (unit-period logit with duration terms). (2) any two-cohort/two-period observational contrast REQUIRES, beyond the raw contrast: ONE covariate/fixed-effects-adjusted model (statsmodels OLS/Logit/GLM with pre-registered covariates drawn from REAL corpus columns — cohort+field+year fixed effects count as adjustment), ONE placebo/falsification test (pre-period cohort contrast, shuffled labels, or a placebo break year), and an explicit inference statement naming the cluster unit (cluster-robust cov_type or grouped bootstrap whenever units repeat within institutions/fields). (3) concentration/inequality indices (Gini/HHI/top-shares) are never compared raw across different sample sizes — use a null-model baseline or a size-corrected index. Each required analysis lands in its own tables/*.csv. If the corpus genuinely cannot support one (no covariate columns, <5 clusters, no pre-period), state that explicitly in notes/data_provenance — a verified infeasibility disclosed in Limitations is honest; a silent skip is not.

9. Figures MUST follow APA 7: numbered, each cited in text, caption BELOW the figure stating the actual plotted variables/axes/groups. Caption text MUST NOT start with 'Figure N' (the typesetter numbers them).

10. Figure titles/captions must name what the plot actually varies. Use 'sensitivity analysis' only when an analysis assumption or robustness setting is varied; a swept treatment level or effect-by-level plot is a dose-response/level-contrast/sweep plot, not a sensitivity analysis.

11. Produce >=3 figures and >=2 tables; at least ONE figure must be a NON-TRIVIAL chart type (not merely a bar/line/scatter/box/histogram/heatmap).

12. Show the distribution/spread (boxplot/violin/per-unit scatter with CIs), never one bar per condition. Never report p==0 (floor it) or an implausible effect size.

13. SURVIVAL-WINDOW COMPARABILITY (cohort designs; mechanically checked): any survival/persistence/renewal outcome compared across birth cohorts MUST use an EQUAL fixed-length post-birth/post-entry window for every cohort — NEVER survival to a common calendar end date (earlier cohorts get more time at risk, which fabricates cohort differences via censoring). Report time-at-risk per analytic cell and declare data_provenance.observation_window = {"window_years": N}.

14. COMPUTE ONCE, REUSE EVERYWHERE: any specification that appears in multiple outputs (primary result, sensitivity tables, figures) must be computed exactly once and the identical estimate/CI/p reused verbatim; re-running a fresh bootstrap for the same cell makes the paper contradict itself via Monte Carlo noise (live defect: text p=0.298 vs table p=0.334 for the same specification).

15. NO DECORATIVE OR SELF-VALIDATION FIGURES in the main set: every main figure must carry a specific inferential or descriptive claim the text argues from. Synthetic calibration/power sweeps and 'sampled network' illustration graphs belong in supplementary material, never beside empirical results. Every table must be self-explanatory: labeled columns with units and a caption naming the variable — a two-column table of bare numbers (e.g. unlabeled year means) is a defect.
